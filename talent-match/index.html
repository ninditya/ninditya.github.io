01 — Problem

Succession planning done by gut. Can data do better?

Company X needs to fill vacancies by matching candidates to the profile of their proven top performers. Without a systematic method, managers rely on subjective judgment — missing employees who look different on paper but share the same underlying success traits.

The task: analyze what actually differentiates Rating-5 employees from everyone else, then encode that into a scoring system any manager can use.

The dataset spans 6 tables, 2,010 employees, and 5 years of assessment data:

  • employees — 2,010 records with org structure (grade, directorate, tenure)
  • performance_yearly — 10,050 rows: ratings 1–5 across 2021–2025
  • profiles_psych — Psychometric scores: IQ, GTQ, TIKI, Pauli, Faxtor, DISC, MBTI
  • papi_scores — 40,200 rows: 20 PAPI work preference scales per employee
  • competencies_yearly — 100,500 rows: 10 competency pillars × 5 years
  • strengths — 28,140 rows: top-14 CliftonStrengths themes per employee

02 — Success Pattern Discovery

What actually separates top performers from everyone else?

187 employees have Rating=5 out of 1,975 valid records (9.5%). The EDA asked one question: across all dimensions, what do these 187 people have in common that the other 1,788 don't?

Competency Execution — the strongest signal

All 10 competency pillars show significant positive differences for R5 employees. The top four differentiators point to a success profile built around people-orientation and business impact, not just technical execution.

The counterintuitive PAPI finding

R5 employees score lower on the PAPI Leadership (G) scale (−0.35), but higher on Planning (Papi_P, +0.35) and Need to Belong (Papi_W, +0.12). High performers at this grade level drive results through collaboration and planning discipline — not positional authority.

Behavioral strengths: future-oriented, self-driven

The top CliftonStrengths themes in R5 — Futuristic (19.8%), Restorative (19.8%), Intellection (18.7%), Self-Assurance (17.6%), Activator (16.0%) — paint a consistent picture: visionary problem-solvers who take initiative without waiting for permission.

DISC type shows no single dominant pattern in R5. High performance is execution-dependent, not style-dependent.

The Success Formula

Based on the EDA, the weighted TGV formula:

Final Match Rate =
  0.30 × Cognitive Ability     -- Pauli & GTQ signal
+ 0.25 × Competency Execution  -- strongest differentiator
+ 0.20 × Work Preferences      -- planning & achievement
+ 0.15 × Behavioral Strengths  -- theme cluster signal
+ 0.10 × Contextual Fit        -- tenure, grade (weak signal)
03 — SQL Matching Algorithm

17 CTEs. One query. Every employee scored.

The matching algorithm is a single parameterized SQL query that accepts a job_vacancy_id and returns every employee ranked by Final Match Rate — no precomputation, no hardcoded IDs.

"The benchmark is never fixed. Any manager can define it at runtime by selecting 1–3 top performers as reference."

CTE Architecture

CTE 1–2
benchmark_employees → unpack selected IDs from talent_benchmarks; benchmark_psych → psychometric scores for those employees
CTE 3–6
baseline_psych, baseline_papi, baseline_competency → PERCENTILE_CONT(0.5) medians per TV across benchmarks
CTE 7–8
benchmark_disc_mode, benchmark_mbti_mode → MODE() WITHIN GROUP for categorical baselines
CTE 9
benchmark_strength_pool → pooled unique top-5 CliftonStrengths themes from all benchmarks
CTE 10–12
candidate_info, candidate_psych, candidate_papi → all 2,010 employees with org context and scores
CTE 13
candidate_strengths_overlap → top-5 theme overlap count per candidate vs benchmark pool
CTE 14
tv_match_rates_long → per-employee × per-TV match rate in long format (UNION ALL across all TVs)
CTE 15–17
tgv_match_rates → AVG within TGV; tgv_weights → hardcoded weights; final_match_rates → weighted total

Scoring formulas

-- Numeric TV (higher = better)
LEAST(100, ROUND(user_score / NULLIF(baseline_score, 0) * 100, 2))

-- Numeric TV (lower = better — inverse PAPI scales Z, K)
LEAST(100, GREATEST(0, ROUND(
    (2 * baseline_score - user_score) / NULLIF(baseline_score, 0) * 100, 2
)))

-- Categorical (DISC, MBTI — exact match)
CASE WHEN user_val = baseline_val THEN 100.0 ELSE 0.0 END
-- NULL when either value missing → no penalty for untested data

-- CliftonStrengths overlap
ROUND(overlap_count::NUMERIC / 5 * 100, 2)

04 — AI App & Dashboard

Any manager can now run a vacancy search.

The Streamlit app wraps the matching algorithm in a form-driven interface: define a vacancy, pick 1–3 top performers as benchmark, and get a ranked list of 2,010 employees — with AI-generated job profiles and interactive visualizations.

What the app does

  • Runtime vacancy creation — Writes a new talent_benchmarks row with a unique vacancy ID. Baselines computed dynamically from selected benchmarks.
  • AI job profile generation — Calls OpenRouter (MiniMax M2.5) with a structured prompt including TGV averages and top CliftonStrengths themes. Returns JSON: job_requirements, job_description, key_competencies.
  • Match score distribution — Histogram of all 2,010 employees, top-15 bar chart, TGV heatmap for top-20 candidates.
  • Candidate detail view — Radar chart (candidate vs benchmark = 100%), TV strengths & gaps bar sorted by match rate, with an 80% threshold line marking competency targets.

05 — Challenges & Learnings

What I would tell myself before starting this project.

ERD vs reality

The case study brief described GTQ1–GTQ5 and Tiki1–Tiki4 as separate columns. The actual database stores single composite values (gtq, tiki). Iterating through the live schema revealed the real structure — documentation can lie.

Model availability

OpenRouter free-tier models deprecated without notice mid-development. Required live API querying to discover a working model. Never hardcode a free-tier model name.

Supabase pagination

PostgREST caps at 1,000 rows per request. Built a custom paginated fetch_table() function to handle all 2,010 employees cleanly — the kind of edge case that only surfaces at scale.

Counterintuitive insights

The PAPI Leadership (G) finding — that R5 employees score lower on leadership style — was the most analytically valuable result. The data pushed back against the obvious assumption, which is exactly when EDA earns its keep.

Ideas for future improvement

Longitudinal competency trend (slope over 5 years) instead of single latest-year score. Role-specific weight sets per job family. Ensemble baseline using performance trajectory weighting. Auto-generated personalized development plans for each candidate.
