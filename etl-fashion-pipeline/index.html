This project is an end-to-end ETL with full unit test coverage using MagicMock.

01 — Problem

Pipelines fail at the edges. The edges you forgot to test.

Most ETL tutorials show the happy path: scrape → clean → load, everything works. Real pipelines fail when HTML structure changes mid-scrape, when a product has a null price, when the database connection drops after 40 successful writes.

The goal of this project was to build an ETL pipeline where every possible failure mode is covered by a test — and where the tests use MagicMock so they run fast, in isolation, without needing a live database or live web server.

02 — Architecture

Three modular layers. Each independently testable.

Extract
utils/extract.py
Web scrapes 50 pages of fashion-studio.dicoding.dev. Per-card parsing with graceful None on broken HTML.
Transform
utils/transform.py
Filters invalid records, converts USD→IDR, parses numerics, strips prefixes, deduplicates.
Load
utils/load.py
Writes to CSV, PostgreSQL (replace), and Google Sheets (clear + update). Each destination has its own error handler.

Extract — safe against broken HTML

Each product card is parsed by extract_product(). The function wraps all field access in a try/except and returns None on any error — so a single broken card never crashes the scrape loop. extract_data() skips failed pages with continue, ensuring partial data is still written rather than nothing.

Transform — explicit validation chain

The cleaning chain is ordered to fail fast: filter null titles and null prices first, then check that price strings actually contain digits (rejects "Price Unavailable"), then convert types. Currency conversion uses an environment variable EXCHANGE_RATE_USD_TO_IDR with a safe default, not a hardcoded constant.

# Explicit validation order — fast fail, no silent drops
df = df[df["title"].notna()]
df = df[df["title"].str.lower() != "unknown product"]
df = df[df["price"].notna()]
df = df[df["price"].str.contains(r"\d", regex=True)]  # reject "Price Unavailable"

EXCHANGE_RATE = float(os.getenv("EXCHANGE_RATE_USD_TO_IDR", 16000))
df["price"] = df["price"].str.replace(r"[$,]", "", regex=True).astype(float) * EXCHANGE_RATE

Load — three destinations, one interface

save_csv(), save_postgres(), and save_gsheet() all follow the same contract: check if DataFrame is empty first, write if not, catch exceptions and log them. Each function is independently callable — the pipeline doesn't fail completely if one destination is unavailable.

03 — Testing Strategy

Every branch covered. No live dependencies.

The test suite uses MagicMock throughout — no real HTTP requests, no real database, no real Google Sheets API. Tests run in milliseconds and pass consistently regardless of network state.

"A test that requires a live database connection is a test that fails at 2am when the database is unreachable."

test_extract.py — mocking HTML structure

Each test creates a mock_card() that simulates a BeautifulSoup div.collection-card using MagicMock.find.side_effect. The mock dispatches on tag name — h3 returns title, span returns price, div returns a detail box with configurable <p> tags.

test_transform.py — data quality contracts

Tests verify that the transform function enforces its contracts on mixed, empty, and all-invalid input:

  • Mixed input — "Unknown Product" and None titles removed; price dtype is float; rating dtype is float; colors dtype is int; size/gender strings stripped of prefixes
  • Empty DataFrame — function returns empty DataFrame without crash (guards against empty extract output)
  • All-invalid input — all records filtered → empty DataFrame returned (not an error)

test_load.py — mocking external systems

save_postgres mocks create_engine and to_sql to verify the correct table name and if_exists="replace" strategy. save_gsheet mocks the full Google API chain — service.spreadsheets().values().clear().execute() and .update().execute() — asserting both are called once.

04 — Design Decisions

Three choices that made the pipeline production-ready.

01
Fail-safe scraping
Each card returns None on error instead of raising. The loop collects partial results — 47 good pages out of 50 is still useful data, not a total failure.
02
Env-var exchange rate
USD→IDR rate reads from EXCHANGE_RATE_USD_TO_IDR env var with a default. Changing the rate never requires a code change — just an env update.
03
Injectable service in load
save_gsheet() accepts an optional service parameter. When provided, it skips credential loading — making the function directly testable without a credentials file.
05 — What this project is really about

QA isn't a safety net. It's a design discipline.

Writing tests for this pipeline changed how I wrote the pipeline itself. Functions that are hard to test are usually hard to maintain. The injectable service parameter in save_gsheet() wasn't planned — it emerged because I needed the function to be testable without credentials.

Testability shapes design

When you write the test first, you catch structural problems early. Functions with too many dependencies are hard to mock — and that difficulty is the design feedback.

MagicMock for external APIs

The Google Sheets API chain (spreadsheets().values().clear().execute()) looks daunting to mock. Chained MagicMock calls just work — each call returns another Mock by default.

Coverage is a floor, not a goal

100% branch coverage doesn't mean the pipeline is correct. It means every code path has been exercised at least once. The actual data quality contracts live in the assertions, not the coverage number.

Multi-destination load

Splitting into save_csv, save_postgres, save_gsheet made each independently testable and independently deployable. A single monolithic save_all() would have been a test nightmare.
